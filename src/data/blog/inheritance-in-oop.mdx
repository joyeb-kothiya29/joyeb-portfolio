## What Does Inheritance Actually Mean?

Inheritance allows one class to reuse the structure
and behavior of another. A new class builds upon an
existing class instead of starting from scratch.
The original becomes the parent. The new class
becomes the child — gaining access to the parent's
features automatically.

## The "Is-A" Relationship

Inheritance only makes sense when there is a natural
relationship between classes. A SportsCar is a Car.
A Car is a Vehicle. A Dog is an Animal. If the
statement sounds logical in real life, it probably
works well in code too.

## Why Developers Use Inheritance

Less Repetition: Define a function once in the
parent — all child classes use it instantly. This
follows the DRY principle: Don't Repeat Yourself.

Easy Updates: Improve a function in the parent and
every child class gets the updated version
automatically — no manual edits across files.

Better Structure: Inheritance organizes related
classes into a clear hierarchy — easier to
understand, extend, and maintain over time.
